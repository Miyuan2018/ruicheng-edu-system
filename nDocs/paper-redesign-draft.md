# 组卷系统改版 — 设计草案

## 确认的设计决策

| # | 决策 | 说明 |
|---|------|------|
| A | 方案2：题目挂单元 → 题型组 | 单元内按题型分组展示，题目挂载到题型组下 |
| B | 同一题不可跨单元复用 | 同一试卷内，一道题只能属于一个单元 |
| C | 单元计时结束 → 自动跳转 | 锁定当前单元，自动进入下一单元 |
| D | 不需要单元间过渡/休息 | 直接跳转 |
| E | 知识覆盖度为"建议" | 软约束，题库不足时自动扩范围 |
| F | 不考虑历史去重 | — |

## 数据模型

### 新增表：`exam_paper_units`

```sql
CREATE TABLE exam_paper_units (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    exam_paper_id UUID NOT NULL REFERENCES exam_papers(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL,              -- 单元名称，如"基础概念"
    description TEXT,                         -- 单元说明（可选）
    position INTEGER NOT NULL DEFAULT 0,      -- 排序
    time_limit_minutes INTEGER,               -- NULL = 不限时
    question_config JSONB NOT NULL DEFAULT '[]',  -- 题型分布配置
    total_score INTEGER NOT NULL DEFAULT 0,   -- 单元总分（计算字段/冗余）
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_units_paper ON exam_paper_units(exam_paper_id, position);
```

### `question_config` JSONB 结构

```json
[
  {
    "question_type": "SINGLE_CHOICE",
    "count": 5,
    "score_per_question": 4,
    "knowledge_points": ["勾股定理定义", "勾股数"],   // 可选，自动选题参考
    "difficulty_ratio": {"EASY": 0.3, "MEDIUM": 0.5, "HARD": 0.2}
  },
  {
    "question_type": "FILL_BLANK",
    "count": 4,
    "score_per_question": 5,
    "knowledge_points": [],
    "difficulty_ratio": {"EASY": 0.2, "MEDIUM": 0.5, "HARD": 0.3}
  }
]
```

### 替换关联表：`exam_paper_questions` → `exam_paper_unit_questions`

```sql
-- 旧表废弃
DROP TABLE exam_paper_questions;

-- 新表
CREATE TABLE exam_paper_unit_questions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    unit_id UUID NOT NULL REFERENCES exam_paper_units(id) ON DELETE CASCADE,
    question_id UUID NOT NULL REFERENCES questions(id),
    question_type VARCHAR(20) NOT NULL,        -- 题型分组标签
    position INTEGER NOT NULL DEFAULT 0,        -- 单元内排序
    score INTEGER NOT NULL DEFAULT 0,           -- 本题分值
    UNIQUE(unit_id, question_id),               -- 保证同单元不重复
    UNIQUE(exam_paper_id, question_id)          -- 保证同试卷不跨单元重复(需触发器或应用层保证)
);

CREATE INDEX idx_upq_unit ON exam_paper_unit_questions(unit_id, position);
```

### ExamPaper 模型变更

| 字段 | 变更 |
|------|------|
| `total_score` | 保留，由各单元 `total_score` 汇总 |
| `duration_minutes` | 保留，可设整卷总时长（与单元计时独立） |
| 新增关系 | `units: List[ExamPaperUnit]` |

## 计时模型（三种模式）

```
试卷 duration_minutes  |  单元 time_limit_minutes  |  行为
───────────────────────┼──────────────────────────┼─────────
NULL                   |  全部 NULL                 |  无计时（练习模式）
有值                   |  全部 NULL                 |  仅整卷计时（传统考试）
NULL / 有值            |  部分/全部有值             |  单元独立计时 + 可选整卷约束
```

单元计时到期 → 自动提交 + 锁定 + 跳转下一单元。最后一个单元到期 → 自动交卷。

## API 端点设计（预览）

```
POST   /exam-papers                          # 创建试卷（元信息）
PUT    /exam-papers/{id}                     # 更新试卷元信息
DELETE /exam-papers/{id}                     # 删除试卷（级联）

# 单元 CRUD
POST   /exam-papers/{id}/units               # 添加单元
PUT    /exam-papers/{id}/units/{uid}         # 更新单元（含 question_config）
DELETE /exam-papers/{id}/units/{uid}         # 删除单元
PUT    /exam-papers/{id}/units/sort          # 单元排序

# 选题（单元内题型组级别）
POST   /exam-papers/{id}/units/{uid}/auto-select    # 自动选题（按 question_config）
POST   /exam-papers/{id}/units/{uid}/questions      # 手动添加题目到单元
DELETE /exam-papers/{id}/units/{uid}/questions/{qid} # 移除题目
PUT    /exam-papers/{id}/units/{uid}/questions/sort  # 题目排序
GET    /exam-papers/{id}/units/{uid}/questions       # 获取单元内题目

# 发布
POST   /exam-papers/{id}/publish             # 发布（不变）
```

### 快速创建行为

| 预设 | 生成结果 |
|------|---------|
| `按题型分组` | 4 个单元：填空题/单选题/多选题/解答题，各含对应题型组，数量均 0 |
| `按知识块分组` | 1 个单元"未命名单元"，无题型，自动展开 |
| `按难度递进` | 3 个单元：基础巩固/能力提升/拓展挑战，无题型，折叠 |
| `空白试卷` | 1 个空单元，折叠 |
| `从模板创建` | 下拉列出 localStorage 保存的单元模板，选中后追加 |

### 交互汇总

| 元素 | 行为 |
|------|------|
| 单元卡片 | 默认折叠，▶ 展开，▼ 折叠 |
| ≡ 手柄 | 拖拽排序单元 |
| × 删除单元 | 确认弹窗，级联删除单元下所有已选题目 |
| `[保存为模板]` | 保存当前单元配置到 localStorage |
| 题型行 | 默认折叠，点击展开编辑难度+知识点 |
| 难度比例输入 | 3 个数字框，单位 %，合计 ≠ 100% 橙色警告 |
| `[+ 选择知识点]` | 弹出知识树面板，多选勾选，支持父子级联 |
| 知识点标签 × | 移除单个知识点 |
| `[+ 添加单元]` | 追加空单元，自动展开 |
| `[+ 添加题型]` | 当前单元追加题型行，默认展开 |
| 底部汇总栏 | 实时计算单元数/题数/总分；整卷限时可设 |

### 校验（下一步时）

- [ ] 至少 1 个单元，名称不为空
- [ ] 每单元至少 1 个题型组
- [ ] 每题型组数量 ≥ 1，分值 ≥ 1
- [ ] 同一单元内题型不重复
- [ ] 难度比例合计 ≠ 100% 橙色警告（不阻止）
- [ ] 知识点溢出蓝色提示（不阻止）

## 页面 3：选题

### 核心交互模型

1. 左侧：单元 → 题型组树，显示选题进度，点击选中"当前目标"
2. 右侧：题库浏览器，筛选条件跟随当前目标（题型自动锁定，学科/年级从试卷继承）
3. 已选题目在题型组下方内联展示，可展开查看
4. 自动选题一键填充所有单元，然后逐题审核替换

### 自动选题逻辑

点击 `[全部自动选题]` 后：

```
对每个单元 → 每个题型组:
  1. 查询候选池: question_type + subject + grade_level 匹配
  2. 知识点加权排序: knowledge_points 中的知识点优先
  3. 难度分层抽样: 按 difficulty_ratio 从各难度档位抽取
  4. 填充题目到题型组
  5. 生成选题报告: 哪些题型组完全满足，哪些不足
```

### 单个题型组的自动选题按钮

每个题型组标题旁有 `[自动选题]` 按钮，仅对该组执行自动选题，不影响其他已选题目。

### 题目卡片操作

| 按钮 | 场景 | 行为 |
|------|------|------|
| `[加入此组]` | 题目未选 | 添加到当前选中的题型组 |
| `[从本组移除]` | 题目已选 | 从题型组移除 |
| `[替换为...]` | 题目已选 | 弹出同类题目列表，选择替换目标 |
| `≡` | 已选题目 | 拖拽调整该组内题目顺序 |
| `×` | 已选题目 | 从题型组移除 |

### 题目详情预览

点击已选题目标题（或题库中题目标题）→ 右侧滑出抽屉，展示完整题目内容（题干、选项、答案、解析），与当前 QuestionEditModal 的预览能力一致。

### 自由浏览模式

题库浏览顶部开关，支持两种模式：

```
● 按目标筛选 (当前模式)    ○ 自由浏览 (显示全部题目)
```

自由浏览模式下不锁定题型/单元，每道题操作按钮变为 `[分配到...]`，点击弹出单元/题型组下拉选择，适合"先挑题再归类"的工作方式。

### 跨单元互斥

已被其他单元选中的题目在题库中标记为不可重复添加：

```
┌──────────────────────────────────────────┐
│ 🔒 题#5: 勾股定理综合应用题               │
│    已在"单元1: 基础概念"中，不可重复选择   │
└──────────────────────────────────────────┘
```

### 题目移动

已选题目卡片增加 `[移动到...]` 按钮，弹出目标单元/题型组下拉，一键跨单元移动。

### 自动选题幂等

- `[全部自动选题]`：仅填充空位，已有选题不动（增量填充）
- `[全部重新生成]`：清空全部 + 重新选题
- `[清空全部]`：清空所有已选题目
- 题型组级 `[自动选题]`：仅对该组执行，已有选题不动
