# V4.3 草稿表 + 保存/发布流程重构

> **目标**：草稿存入独立的 `exam_paper_drafts` 表，与正式试卷主表隔离；保存/发布流程重新设计，状态模型合理化。

**范围**：后端新增草稿表+API，前端 store 改造 autoSave/regenerateAll/saveAll/loadDraft，PaperWizardPage 向导流程，PreviewFinalizeStep 按钮文案

---

## 1. 数据模型变更

### 1.1 新增 `exam_paper_drafts` 表

```sql
CREATE TABLE exam_paper_drafts (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID NOT NULL REFERENCES admins(id),
    paper_id UUID REFERENCES exam_papers(id),  -- 编辑已有试卷时有值，新建为 NULL
    data JSONB NOT NULL,                        -- 完整的 PaperDraft 对象 (Zustand state)
    created_at TIMESTAMP DEFAULT now(),
    updated_at TIMESTAMP DEFAULT now(),
    UNIQUE(user_id, paper_id)                   -- 每人每试卷只有一份草稿
);
```

后端模型：`app/models/exam_paper_draft.py`

### 1.2 `exam_papers.status` 枚举变更

| 旧值 | 新值 | 操作 |
|------|------|------|
| DRAFT | — | **删除**（草稿走独立表） |
| PUBLISHED | PUBLISHED | 保留 |
| ARCHIVED | ARCHIVED | 保留 |
| — | **READY** | **新增**（确认入库但未发布） |

CheckConstraint 更新为 `status IN ('READY', 'PUBLISHED', 'ARCHIVED')`。

### 1.3 Seed 数据和迁移

迁移脚本 `006_v43_drafts.sql`：
- 创建 `exam_paper_drafts` 表
- 将现有 DRAFT 状态试卷迁移到草稿表
- 更新 CheckConstraint

## 2. API 变更

### 2.1 新增草稿 API

文件：`backend/app/api/v1/endpoints/drafts.py`

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/drafts` | 创建/覆盖草稿 |
| GET | `/drafts?paper_id=xxx` | 获取指定试卷的草稿 |
| GET | `/drafts` | 获取当前用户所有草稿 |
| DELETE | `/drafts/{draft_id}` | 删除草稿 |

```python
# POST /drafts body
{
    "paper_id": null,  # 新建时为 null，编辑时为 paper id
    "data": { ... }    # 完整 PaperDraft JSON
}
```

### 2.2 Paper 创建改为存入 READY

`save-all` 端点中 `status` 不再接受 DRAFT，创建时默认 `status='READY'`。

### 2.3 发布端点不变

`POST /{paper_id}/publish` 将 READY → PUBLISHED，逻辑不变。

## 3. 前端 Store 变更

### 3.1 autoSave 改造

**原来**：`autoSave()` → `saveAll()` 写主表

**改为**：`autoSave()` → `paperApi.saveDraft()` 写草稿表

```typescript
autoSave: async () => {
    const { paper, dirty } = get();
    if (!paper || !dirty) return;
    set({ saving: true });
    try {
        // 新建时 paper.id 为 undefined
        await paperApi.saveDraft(paper.id || null, paper);
        set({ saving: false, dirty: false, lastSaved: new Date() });
    } catch {
        set({ saving: false });
    }
},
```

### 3.2 saveAll 改造

**原来**：创建时 `status='DRAFT'`，编辑时发到主表

**改为**：统一发到主表，一律不带 `status`（后端默认 READY）

```typescript
saveAll: async () => {
    const { paper } = get();
    if (!paper || !paper.id) return;
    // ... cleanPaper 不变 ...
    await paperApi.saveAll(paper.id, cleanPaper);
    // 保存成功后删除对应草稿
    try { await paperApi.deleteDraftByPaperId(paper.id); } catch {}
},
```

### 3.3 loadDraft 加强

打开试卷编辑页时，先检查草稿表：

```typescript
loadDraft: async (id: string) => {
    set({ loading: true });
    // 先查是否有未完成草稿
    const draft = await paperApi.getDraft(id).catch(() => null);
    if (draft?.data) {
        // 有待用户确认：使用草稿 vs 从主表加载
        set({ pendingDraft: draft, loading: false });
        return;
    }
    // 从主表加载 ...
},
```

### 3.4 新增 draftApi

```typescript
// frontend/src/api/drafts.ts
export const draftApi = {
    save: (paperId: string | null, data: any) =>
        apiClient.post('/drafts', { paper_id: paperId, data }),
    get: (paperId: string) =>
        apiClient.get(`/drafts?paper_id=${paperId}`),
    getByUser: () =>
        apiClient.get('/drafts'),
    delete: (draftId: string) =>
        apiClient.delete(`/drafts/${draftId}`),
};
```

## 4. PreviewFinalizeStep 变更

### 4.1 按钮区

```
[保存] [保存并发布] | [导出 Word] [导出 PDF] [打印]
```

- `保存`：先 `saveAll`（写主表 READY），再 `deleteDraft`
- `保存并发布`：先 `saveAll`，再 `POST /publish`（READY → PUBLISHED），再 `deleteDraft`

### 4.2 成功提示

| 操作 | 提示 |
|------|------|
| 保存 | `已保存` |
| 保存并发布 | `已保存并发布` |
