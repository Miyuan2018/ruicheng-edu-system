## 目标

手工选题抽屉当前只有关键词搜索，增加**题型**、**难度**、**知识点**三个筛选条件，并支持从当前单元/试卷自动填充初始值。

同时清理知识点存储的两套系统混乱（`meta_data.knowledge_points` + `grade_level.knowledge_points`），统一使用 `grade_level.knowledge_points`（JSONB）。

## UI 设计

### 布局：折叠筛选面板（方案 B）

- 搜索框右侧新增「筛选」按钮，点击展开/折叠筛选面板
- 因打开抽屉时已自动填充了值，**默认展开**让用户确认
- 筛选面板内三个控件横排：题型 Select、难度 Select、知识点 TreeSelect

### 筛选控件

| 控件 | 组件 | 模式 | 自动填充来源 | 空值默认 |
|------|------|------|-------------|---------|
| 题型 | `Select` | 多选 | `unit.question_config[].question_type` 去重 | 全部不选（等同不限题型） |
| 难度 | `Select` | 多选 | `paper.difficulty_ratio` 中比例 > 0 的 key | 全部选中（等同不限难度） |
| 知识点 | `TreeSelect` | 多选 | `paper.knowledge_node_ids`，加载最新版知识树 | 全部不选（等同不限知识点） |

选项枚举值：
- 题型：`SINGLE_CHOICE`, `MULTIPLE_CHOICE`, `FILL_BLANK`, `SUBJECTIVE`
- 难度：`EASY`, `MEDIUM`, `HARD`

### 交互行为

- 筛选条件变更后 **debounce 300ms** 自动重新搜索
- 关键词搜索与筛选条件是 **AND 关系**（叠加过滤）
- 搜索结果仍排除已在试卷中的题目
- 结果卡片补充显示**分数**、**知识点标签**（从 `grade_level.knowledge_points` 读取）

### 防御性检查

- 替换题目前验证 `manualTarget.unitId` 对应的 unit 仍存在于 `paper.units` 中

## 知识点筛选方案（简化版 —— 三轮审阅确认）

### 核心理念

用知识点**文本名称**做筛选参数，不经 UUID 桥接。筛选走 `grade_level.knowledge_points` JSONB 精确 `@>` 匹配。

### 为什么不用 UUID 桥接

- `QuestionKnowledgeNode` 关联表完全为空，无可用数据
- 桥接方案需要 UUID→名称翻译 + 双列 OR，两层间接
- 简化方案：前端直接从 TreeSelect 节点提取 title 传给 API，直观且代码量减半

### 筛选流程

```
TreeSelect 选中节点（value=UUID, label=名称）
  → 前端从 knowledgeNodes 数组提取 title，逗号拼接
  → GET /questions?knowledge_points=一元一次方程,二元一次方程组
  → 后端 grade_level['knowledge_points'] @> ["一元一次方程", "二元一次方程组"]
```

TreeSelect 的 **value 保持 UUID**（节点名称不保证唯一），仅发送请求时提取 title。

### API 参数设计

| 参数 | 类型 | 说明 |
|------|------|------|
| `knowledge_points` | `Optional[str]` | **新增**，逗号分隔的知识点文本名称，JSONB `@>` 精确匹配 |
| `knowledge_point` | `Optional[str]` | **保留**，已有参数（L50），子串 `ilike`，向后兼容 |

两个参数共存，语义互补：`knowledge_points` 做精确限定，`knowledge_point` 做模糊搜索。

### 后端查询实现

```python
# GET /questions 新增参数
knowledge_points: Optional[str] = None  # 逗号分隔的文本名称

if knowledge_points:
    kp_list = [kp.strip() for kp in knowledge_points.split(",") if kp.strip()]
    if kp_list:
        query = query.where(
            Question.grade_level['knowledge_points'].contains(kp_list)
        )
```

生成的 SQL：`grade_level #> '{knowledge_points}' @> '["一元一次方程","二元一次方程组"]'::jsonb`

`@>` 语义：左侧 JSONB 必须包含右侧数组**所有**元素。单列过滤即可覆盖全部题目来源（统一后）。

### 前端 API 调用

```typescript
const params: any = {
  question_type: questionTypes.join(","),
  difficulty: difficulties.join(","),
  knowledge_points: selectedTitles.join(","),  // 从 TreeSelect 节点提取的 title
  subject: livePaper?.subject,
  keyword: keyword || undefined,
  review_status: "APPROVED",
  limit: 50,
};
```

## 实现条件（三轮审阅确认，全部必须）

### C1: 删除 `QuestionBase.knowledge_points` 顶层字段

**问题**: `QuestionBase`/`QuestionCreate`/`QuestionUpdate` 中 `knowledge_points: Optional[List[str]]` 是死代码。前端表单字段名 `knowledge_points_display`（字符串）与 Pydantic 字段名不匹配，`question_in.knowledge_points` 始终为 `None`，`meta_data` 写入从未从前端触发。

**操作**:
- 从 `QuestionBase` 删除 `knowledge_points` 字段
- `create_question` 中移除写 `meta_data` 的代码（L27-29），仅通过 `grade_level` JSONB 写入
- 如仍需 `meta_data.knowledge_points` 展示用，构建为 `meta_data = {"knowledge_points": grade_level.get("knowledge_points", [])}`

### C2: 新参数 `knowledge_points` 与旧参数 `knowledge_point` 共存

**问题**: 已有 `knowledge_point` 参数做子串 `ilike`，改为精确 `@>` 会破坏向后兼容。

**操作**: 新增 `knowledge_points` 参数做 JSONB `@>` 精确匹配，保留 `knowledge_point` 参数的子串行为不变。

### C3: 修复种子脚本，确保 `grade_level.knowledge_points` 有数据

**问题**: 历史数据中 `grade_level.knowledge_points` 可能为 NULL（手工创建 + 旧版本生成）。

**操作**:
- 修复 `seed_v35.py`、`seed_v4.py`，确保所有种子题目写入 `grade_level.knowledge_points`
- `start.sh -d` 跑完种子数据后不再有缺失
- 不在本次做 Alembic migration（避免线上数据迁移风险，种子数据已覆盖演示环境）

### C4: 知识节点列表跨向导步骤共享

**问题**: `BasicInfoStep` 和 `RecommendStep` 都需要加载知识树，重复请求浪费。

**操作**: 将知识节点列表（`knowledgeNodes`）放入 `paperEditor` store 或 React context，Step 1 加载后 Step 3 直接复用，避免重复请求。

### C5: 修复 `update_question` 的知识点写入

**问题**: `update_question`（L553）对 `knowledge_points` 做 `setattr` 会失败（列不存在），静默丢失数据。

**操作**: 若 `question_update.grade_level` 包含 `knowledge_points`，正常写入 JSONB。移除对顶层 `knowledge_points` 的 setattr 调用。如果 `GradeLevel` schema 中还没有 `knowledge_points`，需要添加。
