# V5.02 网络抓取试题改进

**日期**: 2026-06-03
**状态**: 已确认（前端设计通过）
**范围**: 网络抓取试题功能（搜索源、搜索条件、数据格式化）

## 目标

完善 `POST /scrape` 网络抓取试题功能，从三个方向改进：
1. **搜索数据源**：DuckDuckGo → 百度 → 纯 LLM 三级回退
2. **搜索条件**：多值选择 + 批量并行搜索
3. **数据格式化**：四层防护 + LaTeX 预处理

## 1. 搜索数据源：三级回退

```
DuckDuckGo ──失败──→ 百度 ──失败──→ 纯 LLM 生成
```

### DuckDuckGo（优先）

- URL: `https://html.duckduckgo.com/html/?q={query}`
- 反爬宽松，无需 cookie/验证码
- BeautifulSoup 解析 `.result__body` 提取文本片段
- 超时 15s

### 百度（回退）

- URL: `https://www.baidu.com/s?wd={query}`
- 中文教育内容更丰富
- 需携带 Cookie 和真实 User-Agent 模拟浏览器
- 解析 `div.result` 提取摘要文本
- 超时 15s，失败则跳过

### 纯 LLM 生成（兜底）

- 完全不依赖网页搜索
- 直接将搜索条件发给 LLM，利用模型训练知识生成题目
- Prompt 格式与网页搜索后处理一致

### 搜索查询构建

```
query = f"{knowledge_point} {subject} {grade_label} {type_label}"
```

多个知识点/题型/年级时，每个组合生成独立查询字符串分别搜索。

| 字段 | 改前 | 改后 |

### 批量并行搜索

用户选择 n 个知识点、m 个年级、k 个题型时，拆分为 `n × m × k` 个独立搜索任务，`asyncio.gather` 并行执行。

搜索条件预览区显示拆分结果：
```
将拆分 N 个搜索任务:
  知识点: 勾股定理 → 年级: G8 → 题型: 单选题 ×3
  知识点: 勾股定理 → 年级: G8 → 题型: 填空题 ×3
  ...
```

## 3. 数据格式化：四层防护 + LaTeX

### 处理流水线

```
搜索结果文本
  → LaTeX 预处理（format_latex 规范数学表达式）
  → LLM 格式化（强化 Prompt + JSON Schema）
  → JSON 解析
    ├── 成功 → Pydantic 逐字段校验 → 合格 → 入库
    ├── 失败 → 换措辞重试 1 次
    │   ├── 成功 → 校验 → 入库
    │   └── 失败 → JSON 自动修复 → 校验 → 入库/丢弃
    └── 校验失败 → 丢弃 + 记录日志
```

### LaTeX 预处理

使用 `app.utils.latex.format_latex()` 在发给 LLM **之前**规范化数学表达式：

- `a^2` → `a^{2}`（上标包裹花括号）
- `a_2` → `a_{2}`（下标包裹花括号）
- `(x+1)^2` → `(x+1)^{2}`
- 已格式化的 `a^{2}` 保持不变

### Prompt 强化

系统 prompt 中提供：
- 明确 JSON Schema（字段名、类型、必填/选填）
- 一个正确示例
- 常见错误示例及说明

```python
SYSTEM_PROMPT = """你是试题格式化助手。将给定的文本片段格式化为JSON题目数组。

每个题目对象必须包含:
{
  "title": "题目标题（字符串，必填）",
  "question_type": "SINGLE_CHOICE|MULTIPLE_CHOICE|FILL_BLANK|SUBJECTIVE（必填）",
  "difficulty": "EASY|MEDIUM|HARD（必填）",
  "score": 整数（默认5）,
  "correct_answer": "答案：单选题'A'，多选题['A','C']，填空题'答案'，解答题'关键词'",
  "explanation": "解析（字符串）"
}

正确示例:
{"title":"计算3+5=？","question_type":"FILL_BLANK","difficulty":"EASY","score":5,"correct_answer":"8","explanation":"3+5=8"}

错误示例（不要这样做）:
- title不能为空字符串
- correct_answer不能省略
- question_type必须是四个枚举值之一
"""
```

### JSON 解析 + 修复

三层解析策略：
1. 优先匹配 ````json ... ```` 代码块
2. 正则 `\[.*\]` 提取数组
3. 都不行则 JSON 自动修复：
   - 补全末尾缺失的 `]` `}`
   - 修复未闭合的字符串引号
   - 去掉 LLM 可能在 JSON 外输出的解释文字

### 结构化校验

```python
from pydantic import BaseModel, Field

class ScrapedQuestion(BaseModel):
    title: str = Field(..., min_length=1, max_length=500)
    question_type: str = Field(..., pattern="^(SINGLE_CHOICE|MULTIPLE_CHOICE|FILL_BLANK|SUBJECTIVE)$")
    difficulty: str = Field(..., pattern="^(EASY|MEDIUM|HARD)$")
    score: int = Field(default=5, ge=1)
    correct_answer: str = Field(..., min_length=1)
    explanation: str = Field(default="")
```

校验不通过的题目丢弃，记录日志（题目 title + 失败原因），不影响其他合格题目入库。
