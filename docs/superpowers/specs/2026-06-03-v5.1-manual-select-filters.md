# V5.1 手工选题抽屉筛选条件增强

**日期**: 2026-06-03
**状态**: 已确认
**范围**: Step 3 RecommendStep 手工选题抽屉

## 目标

手工选题抽屉当前只有关键词搜索，增加**题型**、**难度**、**知识点**三个筛选条件，并支持从当前单元/试卷自动填充初始值。同时清理旧的知识点文本标签方案残留。

## UI 设计

### 布局：折叠筛选面板（方案 B）

- 搜索框右侧新增「筛选」按钮，点击展开/折叠筛选面板
- 因打开抽屉时已自动填充了值，**默认展开**让用户确认
- 筛选面板内三个控件横排：题型 Select、难度 Select、知识点 TreeSelect

### 筛选控件

| 控件 | 组件 | 模式 | 自动填充来源 |
|------|------|------|-------------|
| 题型 | `Select` | 多选 | `unit.question_config[].question_type` 去重 |
| 难度 | `Select` | 多选 | `paper.difficulty_ratio` 中比例 > 0 的 key |
| 知识点 | `TreeSelect` | 多选 | `paper.knowledge_node_ids`，加载最新版知识树 |

选项枚举值：
- 题型：`SINGLE_CHOICE`, `MULTIPLE_CHOICE`, `FILL_BLANK`, `SUBJECTIVE`
- 难度：`EASY`, `MEDIUM`, `HARD`

### 交互行为

- 筛选条件变更后 **debounce 300ms** 自动重新搜索
- 关键词搜索与筛选条件是 **AND 关系**（叠加过滤）
- 搜索结果仍排除已在试卷中的题目
- 结果卡片补充显示**分数**、**知识点标签**

## API 变更

### `GET /api/v1/questions` 新增参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `difficulty` | `Optional[str]` | 端点已完整实现（L640-641），前端只需传参 |
| `knowledge_point_ids` | `Optional[str]` | **新增**，逗号分隔的 KnowledgeNode UUID |

### knowledge_point_ids 查询实现

通过 `QuestionKnowledgeNode` 关联表过滤（非旧的 `meta_data` JSON 字段）：

```python
if knowledge_point_ids:
    kid_list = [uuid.UUID(k.strip()) for k in knowledge_point_ids.split(",") if k.strip()]
    kn_subq = select(QuestionKnowledgeNode.question_id).where(
        QuestionKnowledgeNode.knowledge_node_id.in_(kid_list)
    )
    query = query.where(Question.id.in_(kn_subq))
```

选择关联表方案的理由：
- 试卷 `knowledge_node_ids` 存的是 KnowledgeNode UUID，与关联表一致
- `meta_data` 列类型是 `JSON`（非 JSONB），不支持 `@>` 运算符
- `meta_data.knowledge_points` 存的是旧文本标签，与 UUID 体系不兼容
- 关联表已有索引，查询高效

## 清理项

### keyword 搜索中的 knowledge_points 文本匹配

`GET /questions` 的 keyword 搜索（L651）对 `grade_level['knowledge_points']` 做 `ilike` 文本匹配 —— 这是合理的全文搜索路径，**保留**。此处 keyword 是自由文本搜索，`knowledge_point_ids` 是结构化 UUID 过滤，两者互补不冲突。
