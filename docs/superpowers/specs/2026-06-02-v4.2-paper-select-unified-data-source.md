# V4.2 选题数据源统一 + 结构调整联动

> **目标**：当教师在 Step2 修改题型数量/分值后，后续步骤始终以试卷结构定义为准；`unit.questions` 作为唯一数据源，多余试题全部标记异常、删除按钮可用；增量补缺口、一键换批、换题去重。

**范围**：前端 `paperEditor` store + `RecommendStep` + `PreviewStep`（分值读取）+ 后端 `auto-generate` 加参数

## 4. RecommendStep 重构 (`steps/RecommendStep.tsx`)

### 4.1 数据源

不再读 `generateReport?.questions`。改为：

```typescript
// 遍历 paper.units，按 question_type 分组
const groupedByType = useMemo(() => {
  const groups: Record<string, ExamPaperUnitQuestion[]> = {};
  paper?.units?.forEach(unit => {
    (unit.questions || []).forEach(q => {
      const qt = q.question_type || 'SINGLE_CHOICE';
      if (!groups[qt]) groups[qt] = [];
      groups[qt].push(q);
    });
  });
  return groups;
}, [paper?.units]);
```

### 4.2 多余检测

```typescript
const isExcess = actualCount > configCount; // 该题型整体是否超量
// 每道题都标记 isExcess（不再只标记尾部）
```

### 4.3 进入 Step3 时自动增量调整

```typescript
useEffect(() => {
  // 检测缺口
  const gaps = [];
  units.forEach(u => {
    (u.question_config || []).forEach(cfg => {
      const existing = (u.questions || []).filter(q => q.question_type === cfg.question_type).length;
      if (cfg.count > existing) gaps.push({ unitId: u.id, questionType: cfg.question_type, deficit: cfg.count - existing });
    });
  });
  if (gaps.length > 0) fillGaps(paper.id);
}, [paper?.id]);
```

### 4.4 一键选题按钮

```typescript
const handleRegenerate = async () => {
  await regenerateAll(paper.id); // 清空所有 → 全量重新生成
};
```

### 4.5 仪表盘

`constraint_dashboard` 从 `generateReport` 中读取，逻辑不变。

### 4.6 换题 & 删除

换题按钮：调用 `swap` API → 用返回的 alternatives 更新对应 question → `replaceQuestion` 同时写入 tags。

删除按钮：`removeQuestionFromUnit(unitId, questionId)` → UI 立即响应（同一数据源）。
